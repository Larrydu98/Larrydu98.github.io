    JS重读【6】—— JavaScript中的面对对象

    DBC 2022/4/8 JS

    主要内容：构造函数、原型和原型链、JS对象继承方式（ES5，ES6）

    Javascript是一种基于对象（object-based）的语言，你遇到的所有东西几乎都是对象。但是，它又不是一种真正的面向对象编程（OOP，Object Oriented Programming）语言，因为它的语法中没有class（类）。

    面向对象编程的特性：

    • 封装：能够将一个实体的信息、功能、响应都封装到一个单独对象中的特性。
      • 由于JavaScript没有public、private、protected这些关键字，但是可以利用变量的作用域来模拟public和private封装特性
    • 继承：在不改变源程序的基础上进行扩充，原功能得以保存，并且对子程序进行扩展，避免重复代码编写，后面的章节详细描述
    • 多态：允许将子类类型的指针赋值给父类类型的指针；原生JS是弱类型语言，没有多态概念

    JavsScript中面向对象的一些概念：

    • 类class： ES5以前就是构造函数，ES6中有class
    • 实例instance和对象object：构造函数创建出来的对象一般称为实例instance
    • 父类和子类：JavaScript也可以称为父对象和子对象

    function objectFactory(){
    var obj = {};
    //取得该方法的第一个参数(并删除第一个参数)，该参数是构造函数
    var Constructor = [].shift.apply(arguments);
    //将新对象的内部属性__proto__指向构造函数的原型，这样新对象就可以访问原型中的属性和方法

    //如果返回值是一个对象就返回该对象，否则返回构造函数的一个实例对象
    return typeof ret === "object" ? ret : obj;
}

    # 2 原型与原型链

    前面提到了，为了解决从原型对象生成实例的问题，Javascript提供了一个构造函数模式，其中，构造函数与实例对象中的prototype和__proto__就是为了构建原型链而存在。

    # 2.1 原型对象

    函数对象有一个的prototype 属性，指向函数的原型对象，它就是一个普通对象。

    • 每个对象都有 __proto__ 属性，但只有函数对象才有 prototype 属性
    • 原型对象都会自动获得一个 constructor（构造函数）属性，指向 prototype 属性所在的函数
    • 实例的构造函数属性（constructor）指向构造函数
    function Person() {};
Person.prototype.name = 'Zaxlct';
Person.prototype.sayName = function() {
  alert(this.name);
}

let person1 = new Person();
let person2 = new Person();

    # 2.2 __proto__

    JS 在创建对象（不论是普通对象还是函数对象）的时候，都有一个叫做__proto__ 的内置属性，用于指向创建它的构造函数的原型对象。

    这个连接存在于实例（person1）与构造函数（Person）的原型对象（Person.prototype）之间，而不是存在于实例（person1）与构造函数（Person）之间

    Person.prototype.constructor == Person;
person1.__proto__ == Person.prototype;
person1.constructor == Person;

    # 2.3 原型链

    简单来说，原型链就是利用原型让一个引用类型继承另一个应用类型的属性和方法。

    以上面的例子来说，

    • 实例person1和person2存在__proto__属性，指向构造函数Person的原型对象Person.prototype；
    • Person.prototype是原型对象，本质上是一个对象，其存在__proto__属性，指向Object.prototype（对象由Object构造函数创建），Function.prototype也是如此；
    • Person是一个函数对象，其存在__proto__属性，指向Function.prototype（实例的__proto__属性指向构造函数的原型对象），Function与Object也是函数对象，其__proto__属性也是指向Function.prototype；
    • Object.prototype也有__proto__属性，指向null，因为这是原型链的最顶端。

    从person1实例的__proto__属性到Person构造函数的原型对象；从Person构造函数的原型对象的__proto__属性，到Function构造函数和Object构造的函数原型对象，这个属性和方法的继承链条就是原型链。

    其原型链如下图所示：

    • 原型和原型链是JS实现继承的一种模型。
    • 原型链的形成是真正是靠__proto__ 而非prototype

    提示

    • Object.__proto__ === Function.prototype
      • Object 是函数对象，是通过new Function()创建的，所以Object.__proto__指向Function.prototype
    • Function.__proto__ === Function.prototype
      • Function 也是函数对象，也是通过new Function()创建，所以Function.__proto__指向Function.prototype。
    • Function.prototype.__proto__ === Object.prototype
      • Function.prototype是个函数对象，理论上他的__proto__应该指向 Function.prototype，就是他自己，自己指向自己，没有意义。 JS一直强调万物皆对象，函数对象也是对象，给他认个祖宗，指向Object.prototype。Object.prototype.__proto__ === null，保证原型链能够正常结束。
    • 原型链是通过__proto__ 进行指向上一级别的prototype，原型链的顶端是Object.prototype
    • 并不是所有的原型都是指向object.prototype,例如

    # 3 ES5原型继承

    # 3.1 原型继承!

    核心：

    • 让子类的原型等于父类的实例

    优点：

    • 原型定义的属性和方法可以复用
    • 简单，易于实现

    缺点：

    • 引用类型的原型属性会被所有实例共享,更改后所有的实例上的的对应的属性都相应的会更改
    • 创建子类实例时，不能向父类的构造函数中传递参数
    function SuperType() {
    this.supername = 'super';
}
SuperType.prototype.getSuperName= function(){
    return this.supername;
}

function SubType () {
    this.subname='subname';
}
SubType.prototype = new SuperType();	// 关键：继承了SuperType
SubType.prototype.getSubName = function (){
    return this.subname;
}

var instance1 = new SubType();
console.log(instance1.getSubName());
console.log(instance1.getSuperName());

    # 3.2 借用构造函数继承!

    核心：

    • 在子类型构造函数中通用call()调用父类型构造函数

    优点：

    • 解决了子类实例共享父类引用属性的问题
    • 创建子类实例时，可以向父类构造函数传参

    缺点：

    • 只能继承父类的实例属性和方法，不能继承原型属性/方法（因为没有用到原型）；
    • 无法实现构造函数的复用，每个子类都有父类实例函数的副本，影响性能，代码会臃肿。
    function SuperType (name) {
    this.name = name;
    this.colors = ['red', 'green'];
}

function SubType (name, age) {
    this.age = age;
    SuperType.call(this, name);	// 关键：继承SuperType
}

var instance1 = new SubType('instance1', 15);
instance1.colors.push('blue'); 
console.log(instance1.colors);  // red, green, blue

var instance2 = new SubType('instance2', 18);
console.log(instance2.colors);  // red, green

    # 3.3 组合继承!

    组合继承=原型继承+构造函数继承。

    核心：

    • 使用原型链实现对原型属性和方法的继承，而通过借用构造函数来实现对实例属性的继承。

    优点：

    • 既通过在原型上定义方法实现了函数复用，又能保证每个实例都有它自己的属性
    • 即解决了原型继承和构造函数继承的缺点

    缺点：

    • 父类构造函数调用了两次
    • 父类中的实例属性和方法既存在于子类的实例中，又存在于子类的原型中（仅内存占用）
    function SuperType (name) {
    this.name = name;
    this.con = [];
}
SuperType.prototype.getName = function() {
    return this.name;
}

function SubType (name, age) {
    SuperType.call(this, name);			// 关键：实现实例属性继承
    this.age = age;
}
SubType.prototype = new SuperType();	// 关键：实现原型属性和方法继承
//SubType.prototype.constructor = SubType;
SubType.prototype.getAge = function() {
    return this.age;
};

var instance1 = new SubType('li', 18);
instance1.con.push('test1');
console.log(instance1.con); // test1
console.log(instance1.getAge()); // 18
console.log(instance1.getName()); // li

var instance2 = new SubType('hang', 18);
console.log(instance2.con); // test1
console.log(instance2.getAge()); // 18
console.log(instance2.getName()); // hang

    # 3.4 原型式继承

    重点：用一个函数包装一个对象，然后返回这个函数的调用，这个函数就变成了个可以随意增添属性的实例或对象。Object.create()就是这个原理，直接将某个对象直接赋值给构造函数的原型。

    优点：

    • 从已有对象衍生新对象，不需要创建自定义类型

    缺点：

    • 原型链继承多个实例的引用类型属性指向相同（所有实例都会继承原型上的属性）；
    • 无法传递参数,无法实现复用（新实例属性都是后面添加的）。
    function object(obj) {
    function O(){}
    O.prototype = obj;
    return new O();

    # 3.5 寄生式继承

    重点：创建一个仅用于封装继承过程的函数，该函数在内部以某种方式来增强对象，最后返回构造函数（就像给原型式继承外面套了个壳子，然后return出来）。

    # 3.6 寄生组合式继承!

    先来看看前面的方式存在的问题：

    • 组合继承解决了传参问题、应用类型共享问题，但又在子类实例构造时，调用父类构造函数两次的问题；
    • 父类型的属性存在两组，一组在实例上，一组在SubType的原型上

    优点：

    • 这种范式的高效率体现在它只调用了一次SuperType构造函数，并且因此避免了在SubType.prototype上面创建不必要的、多余的属性；
    • 原型链能保持不变；能够正常使用instanceof和isPrototypeof()。

    缺点：

    • 实现的过程相对繁琐。
    function inheritPrototype(subType, superType){ 
    var prototype = Object.create(superType.prototype);	// 继承父类的原型
    prototype.constructor = subType; 					// 重写被污染的construct
    subType.prototype = prototype; 						// 重写子类的原型  
}

    # 4.2 class的prototype和__proto__

    # （1）类的prototype属性和__proto__属性

    大多数浏览器的 ES5 实现之中, 每一个对象都有__proto__属性, 指向对一个的构造函数的prototype属性, Class 作为构造函数的语法糖, 同时有prototype属性和__proto__ 属性, 因此同时存在两条继承链

    • 子类的 __proto__属性, 表示构造函数的继承, 总是指向父类.

    • 子类的 prototype 属性的__proto__属性, 表示方法的继承, 总是指向父类的 prototype 属性.

    class A { }
class B extends A { }

B.__proto__ === A // true
B.prototype.__proto__ === A.prototype // true

    5

    上面代码中，子类B的__proto__属性指向父类A，子类B的prototype属性的__proto__属性指向父类A的prototype属性。这样的结果是因为，类的继承是按照下面的模式实现的。

    class A { }
class B { }

Object.setPrototypeOf(B.prototype, A.prototype); // B 的实例继承 A 的实例
Object.setPrototypeOf(B, A);   // B 继承 A 的静态属性
const b = new B();

    这两条继承链，可以这样理解：

    • 作为一个对象，子类（B）的原型（__proto__属性）是父类（A）；
    • 作为一个构造函数，子类（B）的原型对象（prototype属性）是父类的原型对象（prototype属性）的实例。

    # （2）实例的 __proto__ 属性

    子类实例的 __proto__ 属性的 __proto__属性, 指向父类实例的__proto__ 属性, 也就是说, 子类的原型的原型, 就是父类的原型.

    var p1 = new Point(2, 3);
var p2 = new ColorPoint(2, 3, 'red');

p2.__proto__ === p1.__proto__ // false
p2.__proto__.__proto__ === p1.__proto__ // true
