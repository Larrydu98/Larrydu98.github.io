    主要内容：JavaScript的一些基础接口和内容

    # 1. var let const

    # var ——ES5变量声明方式

    • 在变量未赋值的时候，变量underfined
    • 作用域——var的作用域是方法作用域，只要在方法内进行定义了，整个方法内的定义变量后的代码都可以使用
      • 此处详细可以看
    • 可以重复声明

    # let——ES6变量声明方式

    • 在变量声明之前使用会直接报错；
    • 作用域——let是块级作用域，通常let作用域比var的作用域要小
    • let禁止重复声明变量，否则会报错。
    • 在for循环中会形成变量作用域死区

    # const——ES6变量声明方式

    • const是常量声明方式：声明变量时必须初始化，在后面出现的代码也不允许修改该常量
    • 对于引用数据类型来说并不是变量的值不能改动，只是变量指向的那个内存地址不能更改；

    # 2. JS数据类型

    基本数据类型（7大）：Number，String，Boolean，null，underfined, symbol, bigint (后两个是ES6新增)

    • 直接存储在栈中的简单数据段，占据空间小，大小固定，属于被频繁使用的数据

    引用数据类型：

    存储在堆空间，占据的空间大，大小不固定，同时在栈中也存储指针

    • object：普通对象，数组对象，正则对象，日期对象，Math数学对象
    • function:函数对象

    nul和underfined的区别

    • underfined表示一个变量自然地最初始的状态值

      • 申明一个变量但没有赋值
      • 访问对象上不存在的属性
      • 函数定义了形参但是没有传递形参
      • 使用void对表达式求值

    • null：表示一个变量认为设置的空对象，而不是状态，当需要释放一个变量赋值为null

      • Typeof null是一个object是历史遗留问题：

      由于js数据类型在底层是以二进制形式表示的，二进制的前三位为0就会别判断为对象类型，而同时null的二进制恰好是，null被误判为object类型。

    堆和栈的区别：

    • 堆比栈空间大，栈比堆运行速度快
    • 堆空间是一个无序的空间，可以根据引用直接获取；
    • 基础数据类型是比较稳定的，占据的空间较小

    # 3. Object.assign、 map和foreach、indexof、for...of

    # Object.assign

    作用：可以实现对象的合并，并实现浅克隆

    语法：Object.assign(target, sources)

    解析：

    • Object.assign会将source中的可枚举属性都赋值到target，如果target已经有了重复的属性名，则会覆盖
    • 后所需的source会覆盖前面的source同名属性
    • object.assign复制的是属性值，如果属性值是一个引用类型，那么复制的其实就是引用地址，就会存在引用共享的问题
    • 它将返回目标对象。

    # map和forEach的区别

    相同点：

    • 都会循环遍历每一项目
    • 每次执行匿名函数都执行三个参数，分别是item（第一项），index（索引值），arr(原数组）
    • 匿名函数的this都是指向的是windows
    • 只能遍历数组

    不同点：

    • map会分配内存空间存储新的数组并返回，forEach不会返回数据
    • forEach（）的callback更改原始数组的元素，map会返回新的数组

    # for of可以遍历哪些：

    for...of...，是ES6新增的一个遍历方法。但是只限制于迭代器（iterator），所以普通对象对for...of是会报错的，

    可迭代的对象：Array, Map, Set, String, TypedArray， arguments对象。

    # Instanceof

    instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。

    语法：

    object instanceof constructor

    # 4. Map和Object

    # Map和Object区别

    Objects (opens new window) 和 Maps 类似的是，它们都允许你按键存取一个值、删除键、检测一个键是否绑定了值。因此（并且也没有其他内建的替代方式了）过去我们一直都把对象当成 Maps 使用。不过 Maps 和 Objects 有一些重要的区别，在下列情况里使用 Map 会是更好的选择：

    Map Object
    意外的键 Map 默认情况不包含任何键。只包含显式插入的键。 一个 Object 有一个原型, 原型链上的键名有可能和你自己在对象上的设置的键名产生冲突。
    **备注：**虽然 ES5 开始可以用 Object.create(null) 来创建一个没有原型的对象，但是这种用法不太常见。
    键的类型 一个 Map的键可以是任意值，包括函数、对象或任意基本类型。 一个Object 的键必须是一个 String (opens new window) 或是Symbol (opens new window)。
    键的顺序 Map 中的 key 是有序的。因此，当迭代的时候，一个 Map 对象以插入的顺序返回键值。 一个 Object 的键是无序的
    **备注：**自ECMAScript 2015规范以来，对象确实保留了字符串和Symbol键的创建顺序； 因此，在只有字符串键的对象上进行迭代将按插入顺序产生键。
    Size Map 的键值对个数可以轻易地通过size (opens new window) 属性获取 Object 的键值对个数只能手动计算
    迭代 Map 是 iterable (opens new window) 的，所以可以直接被迭代。 迭代一个Object需要以某种方式获取它的键然后才能迭代。
    性能 在频繁增删键值对的场景下表现更好。 在频繁添加和删除键值对的场景下未作出优化。

    # 5.6 利用hasOwnProperty

    所有的都可以实现去重

    function unique(ary) {
  let obj = {};
  return ary.filter(function (item, index, a) {
    // item : 数组每一个成员
    // index: 成员对应的索引
    // a : 整个数组
    // hasOwnProperty来校验的该属性是否出现过；
    // return obj.hasOwnProperty(typeof item + item) ? false : obj[typeof item + item] = true;
    if (obj.hasOwnProperty(typeof item + item)) {
      return false
    } else {
      obj[typeof item + item] = true;
      return true;
    }
  })
}
//[1, true,'true', 15,false, undefined, null, NaN, {}, 'NaN', 'a']，NaN可以去去除但是{}空对象不行

    # 5.9 递归

    function unique(ary) {
  let len = ary.length;
  ary = ary.sort(function (a, b) {
    return a - b;
  });

  function loop(index) {
    if (index >= 1) {
      if (ary[index] === ary[index - 1]) {
        ary.splice(index, 1);
      }
      loop(index - 1)
    }
  }
  loop(len - 1);
  return ary;
}
//对象和NaN没法去除

    # 6. 数组与伪数组

    **数组：**是一个特殊的对象与常规对象有区别

    • 当有新元素添加到列表中，自动更新length属性
    • 设置length属性可以进行截断数组
    • 方法继承于Array.prototype

    **类数组：**是一个拥有length属性没并且其他属性为非负整数的普通对象，类数组不能调用数组的方法

    • 函数参数arguments，
    • DOM对象列表(例如document.querySelectorAll('div'))，
    • jQuery中的$('div')等

    类数组转化为数组：

    • 转换方法

      • Array.from()
      • Array.prototype.slice.call()
      • Array.prototype.forEach()进行属性遍历形成新的数组

    • 转换须知

      • 转换后的数组长度由length决定，索引不连续时转换结构是连续的自动补位

    • 代码示例

      • //仅考虑0或者正整数的索引，对于索引不连续的也是直接进行补位
let obj = {
  '-1': -1,
  0: 0,
  1: 1,
  2: 2,

  3: 3,
  5:5,
  'length': 6

    # 7. JS类型判断方法

    1. # typeof检测变量是否是number, string, boolean, function, undefined, object，bigint， symbol
      • 对于基本类型，除 null 以外，均可以返回正确的结果。
      • 对于引用类型，除 function 以外，一律返回 object 类型。
      • 对于 null ，返回 object 类型。
      • 对于 function 返回 function 类型。
    2. # instanceof来判断 A 是否为 B 的实例，表达式为：A instanceof B，如果 A 是 B 的实例，则返回 true,否则返回 false。 在这里需要特别注意的是：instanceof检测的是原型，
      • instanceof 只能用来判断两个对象是否属于实例关系， 而不能判断一个对象实例具体属于哪种类型。
      • 针对数组的这个问题，ES5 提供了 Array.isArray() 方法 。该方法用以确认某个对象本身是否为 Array 类型，而不区分该对象在哪个环境中创建。
    3. # constructor 判断数据类型的语法是 target.constructor === constructor。 示例如下：
      // Number
console.log((123).constructor)

console.log('123'.constructor)
